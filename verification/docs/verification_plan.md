# AXI Crossbar 验证计划

## 1. 概述

本文档描述AXI Crossbar模块的完整验证计划，遵循企业级数字IC验证流程。

### 1.1 设计规格

| 参数 | 值 |
|------|-----|
| 主接口数量 | 4 |
| 从接口数量 | 4 |
| 地址宽度 | 16-bit |
| ID宽度 | 8-bit |
| 数据宽度 | 32-bit |
| AXI信号模式 | AXI4 |

### 1.2 地址映射

| 从接口 | 起始地址 | 结束地址 | 大小 |
|--------|----------|----------|------|
| SLV0   | 0x0000   | 0x0FFF   | 4KB  |
| SLV1   | 0x1000   | 0x1FFF   | 4KB  |
| SLV2   | 0x2000   | 0x2FFF   | 4KB  |
| SLV3   | 0x3000   | 0x3FFF   | 4KB  |

### 1.3 主接口ID掩码

| 主接口 | ID掩码 | ID范围 |
|--------|--------|--------|
| MST0   | 0x00   | 0x00-0x0F |
| MST1   | 0x10   | 0x10-0x1F |
| MST2   | 0x20   | 0x20-0x2F |
| MST3   | 0x30   | 0x30-0x3F |

## 2. 验证策略

### 2.1 验证方法学

- **UVM-like方法**: 采用基于SystemVerilog的验证方法学
- **覆盖率驱动**: 功能覆盖率 + 代码覆盖率
- **断言验证**: SVA (SystemVerilog Assertions)
- **直接测试 + 随机测试**: 结合定向测试和受约束随机测试

### 2.2 验证层次

```
Level 1: 单元验证 (Unit Verification)
  - Round Robin仲裁器
  - 地址解码器
  - Pipeline模块
  
Level 2: 模块验证 (Block Verification)
  - Slave Switch (读/写通道)
  - Master Switch (读/写通道)
  - Slave/Master Interface
  
Level 3: 系统验证 (System Verification)
  - 完整Crossbar功能
  - 多主多从并发
  - 协议合规性
```

## 6. 验证组件

### 6.1 AXI Master Agent
- **Driver**: 生成AXI主接口事务
- **Monitor**: 监控AXI主接口活动
- **Sequencer**: 产生事务序列

### 6.2 AXI Slave Agent
- **Driver**: 响应AXI从接口请求
- **Monitor**: 监控AXI从接口活动
- **Memory Model**: 模拟从接口存储

### 6.3 Scoreboard
- **地址路由验证**: 检查事务是否路由到正确从接口
- **数据完整性**: 验证读写数据一致性
- **顺序检查**: 确保事务顺序正确

### 6.4 Coverage Collector
- **功能覆盖率**: 收集预定义覆盖点
- **协议覆盖率**: 监控AXI协议特性

## 7. 运行计划
